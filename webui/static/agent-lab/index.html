Experimental Agent Lab

Import, configure, run, register, and test real AI agents

Use Git import, ZIP upload, local folder upload, or mapped project folders. Configure Ollama, LM Studio, OpenRouter, or custom OpenAI-compatible providers, then run the agent in Docker with optional GPU access and auto-create VulnoraIQ scan targets.

Main Console

1. Import Agent

Import real source code. No mock/demo projects are created by this interface.

Select a local AI-agent source folder. The browser uploads it into managed Agent Lab storage under ./agent-lab/projects/; VulnoraIQ never receives a raw local filesystem path.

Desktop Mode creates ./projects/ beside the launcher. Place a real AI agent in ./projects/<agent-name>/, then refresh. Managed imports live in ./agent-lab/projects/.

Mapped projects are read-only. If a mapped project has no Dockerfile, either add one to that folder or upload/import it into managed Agent Lab so VulnoraIQ can generate one.
