# Security Policy

## Responsible use

This framework is intended for authorised security testing, internal validation, and defensive AI red-team exercises. Do not run it against third-party systems without written permission.

## Reporting vulnerabilities in this project

Please open a GitHub security advisory or private issue if you identify a vulnerability in the framework itself.

Include:

- Affected version or commit
- Reproduction steps
- Impact
- Suggested mitigation if known

## Payload safety policy

Payloads in this repository should be safe starter examples. Contributions that enable real-world credential theft, malware deployment, or harmful bypass workflows will not be accepted.
